# Claude Code vs. Cowork — when to use which

> **Archived slide.** This content was previously a slide in `slides.html`
> ("Session basics" section). It was removed from the deck to keep the
> workshop focused on the "files, not chats" thesis, and preserved here for
> reference. Not maintained as part of the live deck.

|                | Claude Code                          | Cowork mode                                        |
|----------------|--------------------------------------|----------------------------------------------------|
| **Surface**    | Terminal CLI (also IDE plugins)      | Desktop app feature                                |
| **Centered on**| A project folder of code/data        | Cross-tool workflows on your computer              |
| **Reads / writes** | Files, runs shell commands       | Files + connectors (Slack, Gmail, Notion, Calendar)|
| **Audience**   | Developers, researchers              | Knowledge workers; non-developers                  |
| **Strong fit** | Solver code, experiments, paper LaTeX| Grant admin, email triage, document pipelines      |
| **Shared**     | Same model family, plugins, skills, memory conventions — across both. ||

For the kind of computational research this workshop covers, Claude Code is the
workhorse. Cowork is useful when the work crosses tools your terminal can't
reach — drafting a referee reply that pulls from your Calendar, Gmail, and a
Notion outline at once.
