# Plan: FilterADMM walkthrough notebook + baseline comparison

## Context

`admm_walkthrough.ipynb` is the existing teaching notebook for the **baseline**
`admm.py` driver: 17 cells, six sections building from compliance → ADMM math →
each subproblem → full ADMM loop → α-sweep. `STATUS.md` "What's next" item 3
flags adding a `FilterADMM.py` walkthrough as the next documentation task.

This plan creates a parallel notebook **`filter_walkthrough.ipynb`** that:

1. Motivates the filter (baseline ADMM oscillates near convergence on the
   30×10 problem).
2. Walks through the new primitives `Filter`, `compute_eta`,
   `compute_omega`, `project_box_budget`, and the restoration phase.
3. Runs `filter_admm_compliance_tv` end-to-end with the same plotting style
   as the existing PDF outputs (`images/convergence.pdf`, `images/iter_NNNN.pdf`)
   but inline.
4. Ends with a head-to-head comparison: same problem, same `eps_opt`, both
   drivers — showing the baseline failing to converge where FilterADMM does.

Following the existing pattern, the notebook is generated by a Python script
(`generate_filter_notebook.py`, sibling to `generate_notebook.py`) — the
`.ipynb` file is never edited by hand.

## Notebook outline (`filter_walkthrough.ipynb`)

All 30×10 throughout for a fair head-to-head with the documented baseline

volume fraction, α=0.5, ρ_init=1, eps_opt=1e-2 — same parameters as
`FilterADMM.py`'s `__main__` self-test.

### Title + setup (3 cells)

- **Title md.** One paragraph: states the same TopoOpt problem as
  `admm_walkthrough` (with the same math block), then says "this notebook
  extends the baseline ADMM with an augmented-Lagrangian filter and
  adaptive ρ, fixing the convergence failures of the fixed-ρ baseline".
  Has a "see also `admm_walkthrough.ipynb` for the x/y subproblem solvers
  used as black boxes here" pointer.
- **Imports code.** `numpy`, `matplotlib`, `mcolors`. From the project:
  `ComplianceProblem`, `admm_compliance_tv`, `build_grid_edges`,
  `filter_admm_compliance_tv`, `Filter`, `compute_eta`, `compute_omega`,
  `project_box_budget`. Same `plt.rcParams` setup as the baseline notebook.
- **Parameters code.** `nelx=30, nely=10, penal=3.0, volfrac=0.4,
  budget=120.0, alpha=0.5, rho_init=1.0, eps_opt=1e-2, x_lo=1e-4`. Print
  banner.

### §1 — Motivation: why a filter? (1 md + 1 code = 2 cells)

- **Md.** Brief recap (one paragraph) of the ADMM splitting; refer reader
  to `admm_walkthrough §2` for the math derivation. State that fixed ρ
  oscillates near convergence; quote the documented 30×10 failure (iter
  27→28: η jumps 0.012 → 0.85; ω jumps 0.005 → 0.12).
- **Code.** Run baseline `admm_compliance_tv` on 30×10 with `max_iter=40`
  (so we see the post-iter-27 oscillation). Store `info_admm` for reuse in
  §3 (computing η/ω along the trajectory) and §6 (comparison).
  Plot primal and dual residuals on log-y axis, mark iter 27 with a
  vertical line. Annotate the jump.

### §2 — The augmented-Lagrangian filter (1 md + 1 code = 2 cells)

- **Md.** Acceptance condition `\eqref{eq:filter}`: a point $(η,ω)$ is
  accepted iff for every existing entry $(η_l, ω_l)$ either $η ≤ β·η_l$ OR
  $ω ≤ ω_l - γ·η$. Dominance pruning (an entry dominated on both axes is
  removed). Pointer to `RASC-MeetingNotes §3.1`.
- **Code.** Demo: create `Filter(beta=0.99, gamma=1e-5)`, add 3 contrived
  entries, then plot the acceptance region in (η, ω) space:
  - Filter entries as black dots.
  - Rejected region shaded grey (union of "L"-shaped corners per entry).
  - 6 test points: green ✓ for accepted, red ✗ for rejected, with
    `is_acceptable` calls printed underneath as a sanity check.

### §3 — Optimality measures η and ω̃ρ (1 md + 1 code = 2 cells)

- **Md.** Definitions of `η = ‖x-y‖²` `\eqref{eq:eta}` and `ω̃ρ`
  `\eqref{eq:omega-prox}` — the proximal-gradient extension for the
  non-smooth `α·TV(y)` term. Brief explanation that the literal
  projected-gradient `ω_ρ` is undefined for non-smooth y, so the y-block
  uses a prox-step instead. Mention the §3.3 identity:
  $\widetilde{ω}_ρ(x^{k+1},y^{k+1},λ^k) = \widetilde{ω}_0(x^{k+1},y^{k+1},λ^{k+1})$
  after the multiplier update.
- **Code.** Along the baseline trajectory stored in §1's `info_admm`:
  reconstruct (x, y, λ) at each iteration from `info_admm` (the baseline
  driver stores `y`, `lam`; we recompute `x` from saved snapshots — or, if
  not available, re-run with a callback). Compute η and ω̃ρ at each. Plot
  on log-y axis, alongside the primal/dual residuals from §1. The point:
  η and ω don't converge — the residual-based stopping criterion gave up
  before the *real* first-order measures stabilised.
  *Implementation note:* `admm.admm_compliance_tv` doesn't currently
  store x-trajectories. Easiest: re-run with an `outer_callback` that
  appends `x.copy()` to a list, then compute η/ω post-hoc. (No edit to
  `admm.py` — the callback hook already exists per CLAUDE.md.)

### §4 — Restoration phase (1 md + 1 code = 2 cells)

- **Md.** Three restoration triggers:
  1. eq.~(8): $η ≥ β·U$ (large primal infeasibility)
  2. eq.~(9): $ω ≤ ε ∧ η ≥ β·η_{\min}$ (dual converged but primal stale)
  3. $j+1 = M_{\max}$ (inner-iteration cap exhausted)
  Backtracking $\hat y = x + 2^{-m}(y - x)$ shrinks η geometrically toward
  0; the first filter-acceptable $m$ is accepted, and ρ is doubled.
  Briefly enumerate the four departures + ω̃₀ correction with pointers to
  `RASC-MeetingNotes §4.1–§4.5`.
- **Code.** Contrived demo: take an (x, y) pair with `η ≈ 1` from the
  baseline run (e.g. iter 5 from §1's info), pretend it's a candidate.
  Compute η for the original (x, y) and for the backtracked $\hat y$ at
  $m=1,2,3,4,5$. Plot η vs m on log-y axis showing the geometric decrease.
  Also show that the filter (with the entries from §2) rejects the
  original but accepts $\hat y$ at some $m^* > 0$.

### §5 — Full FilterADMM (1 md + 3 code = 4 cells)

- **Md.** Algorithm overview: outer loop on $(x, y, λ, ρ)$; inner ADMM
  loop until filter-acceptable; restoration on triggers. Pointer to
  `RASC-MeetingNotes §4.6` Algorithm 2 for the formal statement. Brief
  list of the 4 departures + ω̃₀ correction with §-references (no need to

- **Code 1.** Run
  `x_opt, info = filter_admm_compliance_tv(..., verbose=False, verbose_inner=False, plot=False)`

- **Code 2.** Plot the convergence trajectory: 3 subplots side-by-side

  - Middle: ω vs outer iter, log-y (red squares).
  - Right: ρ vs outer iter, step plot (purple), with R-markers at the

### §6 — Comparison with baseline (1 md + 2 code = 3 cells)

- **Md.** Setup: same problem (30×10, 40%, α=0.5, eps_opt=1e-2), both

## Post-implementation notes (added 2026-05-18 after the run)

All verification steps from the plan passed. The implementation matched
the plan structure exactly (18 cells, six sections, single 30×10 mesh
throughout). Two issues caught during execution and fixed:

1. **`info["n_iter"]` vs `len(info["eta_history"])` off-by-one.** In
   FilterADMM, the convergence check at the top of the outer loop
   `break`s before the per-iteration history appends — so `n_iter` is
   one more than the history length. Plot cells using `np.arange(1,
   info["n_iter"]+1)` for the x-axis mismatched the y-axis array lengths
   and crashed in `ax.semilogy`. Fixed by using
   `np.arange(1, len(info["eta_history"]) + 1)`. The baseline `admm.py`
   does not have this off-by-one (its appends happen *before* the break
   check), so §1 still uses `info_admm["n_iter"]` as written.

2. **`Filter.entries` is private.** Initial draft used `flt.entries` for
   the §2 acceptance-region plot. The Filter class exposes entries via
   `__iter__` instead (`for e, w in flt:` or `list(flt)`); the private
   `_entries` attribute isn't part of the contract. Fixed by switching
   to iteration.

Final run on the 30×10 default (executed via
`jupyter nbconvert --execute`):

- Notebook: 18 cells, 11 code + 7 markdown, 7 inline PNG plots, no cell
  errors. 741 KB on disk.
- Baseline ADMM in §1: 40 iterations (max_iter cap), did not converge.
  Iter-27 oscillation visible in the residual plot exactly as documented.
- FilterADMM in §5: 14 outer iterations, converged, 3 restorations
  (ρ: 1→2→4→8), η=3.6e-3, ω=7.8e-3, objective 331.45 — matches the
  post-`ω̃₀`-correction numbers in `RASC-MeetingNotes.tex` §4.6.
